Modyfikowanie template'u

Należy dodać wsparcie dla WooCommerce w motywie.

add_action( 'after_setup_theme', 'woocommerce_support' );
function woocommerce_support() {
add_theme_support( 'woocommerce' );
}

Modyfikując wygląd elelemntu WooCommerce, należy skopiować tylko plik za to odpowiadający do katalogu /woocommerce w katalogu motywu/motywu potomnego, np

/plugins/woocommerce/templates/single-product/related.php do → /themes/katalog_motywu/woocommerce/single-product/related.php

Formatowanie ceny

Należy zawsze używać funkcji wc_price() oraz wc_format_sale_price() posiadając cenę w formie zmiennej np. float lub integer, dzięki temu zachowana zostanie spójność z formatowaniem w całym sklepie oraz możliwe będzie zmiana formatowania z poziomu ustawień WooCommerce.

Importowanie produktów

Najefektywniejszym mechanizmem importowania produktów jest funkcja wp_insert_post().

wp_post_insert(
    array(
        'post_author'           => $user_id,
        'post_content'          => '',
        'post_content_filtered' => '',
        'post_title'            => '',
        'post_excerpt'          => '',
        'post_status'           => 'draft',
        'post_type'             => 'post',
        'comment_status'        => '',
        'ping_status'           => '',
        'post_password'         => '',
        'to_ping'               => '',
        'pinged'                => '',
        'post_parent'           => 0,
        'menu_order'            => 0,
        'guid'                  => '',
        'import_id'             => 0,
        'context'               => '',
        'post_date'             => '',
        'post_date_gmt'         => ''
        'tax_input'    => array(),
        'meta_input'   => array(),
    )
)

Należy starać się tworzyć post_meta (meta_input) i term (tax_input) na etapie tworzenie tego produktu.

cart_fragments

WooCommerce Cart Fragments to API zaprojektowane w celu zmniejszenia ilości danych przesyłanych między przeglądarkami a serwerami podczas typowych sesji zakupowych np. odświeżanie koszyka. Odbywa się to poprzez przechowywanie pewnych zmian w "fragmentach" - małych fragmentach danych — które są następnie ponownie wysyłane do przeglądarek w razie potrzeby, zamiast zmuszać je do ponownego załadowania całej strony. Proces ten może poprawić wydajność, zmniejszając obciążenie serwera i opóźnienia spowodowane przeładowaniem całej strony.

Przykładowe użycie — funkcja automatycznie aktualizująca ilość produktów w koszyku przy ikonie koszyka.

add_filter( 'woocommerce_add_to_cart_fragments', array( $this, 'wc_refresh_mini_cart_count' ) );
public function wc_refresh_mini_cart_count( $fragments ) {
  ob_start();
  $this->mini_cart_counter();
  $fragments['#count-products-cart'] = ob_get_clean();
  return $fragments;
}
public function mini_cart_counter() {
  $count = intval( WC()->cart->get_cart_contents_count() );
  $class = 'counter';
  if ( $count > 0 ) : 
    $class = $class . ' is-visible';
  endif;
  ?> 
  <span id="count-products-cart" class="<?php echo esc_attr( $class ); ?>">
    <?php echo esc_html( $count );?>
  </span>
  <?php
}

Przy ikonce koszyka należy dodać: $this->mini_cart_counter(); - ta metoda odpowiada z wyświetlanie ilości produktów w koszyku. Inne dostępne metody api:

  • woocommerce_add_to_cart_fragments,
  • woocommerce_update_order_review_fragments,
  • woocommerce_update_order_review_fragments.

Reponsywność koszyka

Podstwowym problemem występującym w koszyku jest użycie tabelek, przez co jego widok nie jest mobilny i nie może być responsywny. W tym wypadku należy podmienić plik cart w katalogu katalog_motywu/woocommerce/cart/cart.php
